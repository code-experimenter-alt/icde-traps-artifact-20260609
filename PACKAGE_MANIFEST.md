# TRAPS Submission Package Manifest

Prepared for the ICDE submission.

## Submission Files

- `main.pdf`: compiled paper. Current local build is 13 pages total, with the 12-page main body ending before the AI-generated content acknowledgement and references.
- `packages/ICDE_TRAPS_overleaf_minimal.zip`: minimal Overleaf/LaTeX package containing `main.tex`, `references.bib`, `main.bbl`, all included `context/*.tex` files, and the figures used by the main paper.
- `packages/ICDE_TRAPS_github_code.zip`: full artifact package containing the paper source, generated result CSVs, experiment scripts, used figures, artifact guide, executable audit, and this manifest.
- `packages/SHA256SUMS.txt`: SHA-256 checksums for `main.pdf` and both submission zip files. This file is regenerated by `python experiments/package_submission.py` after the zip files are written.
- `SUBMISSION_ARTIFACT_UPLOAD.md`: reviewer-facing upload checklist and suggested artifact metadata.

Current author mode: the compiled paper and source package include the full author block. ICDE 2027 uses single-blind review, so this is the intended submission mode.

## Full Code Package Contents

The full code package is intentionally narrow. It includes:

- top-level paper and artifact documentation,
- `context/*.tex`,
- `experiments/run_icde_experiments.py`,
- `experiments/run_lanl_trace_experiment.py`,
- `experiments/audit_submission_requirements.py`,
- `experiments/package_submission.py`,
- `experiments/results/*.csv`, including the LANL status-outcome replay and dynamic drift-guard summaries,
- ICDE-facing experiment figures under `figure/icde_experiments/`,
- the learned Bloom/filter schematic used by the paper.

It excludes local scratch directories, Python bytecode, rendered dashboard scratch files, and the old dashboard PDFs that are no longer used by the main paper.

## Reproduction Gates

Run these commands from the repository root:

```powershell
python experiments/run_icde_experiments.py
$env:LANL_AUTH_INSECURE='1'
python experiments/run_lanl_trace_experiment.py
latexmk -pdf -interaction=nonstopmode -halt-on-error main.tex
python experiments/package_submission.py
python experiments/audit_submission_requirements.py
```

The audit must pass after packaging. It verifies the experiment grids, exact-tag/Bloom substrate framing, dynamic drift-guard evidence, route-coverage and fail-to-backend language, denominators, cache invariants, LANL negative controls, queueing outputs, LaTeX log quality, package cleanliness, and checksum freshness.

## External Upload Requirement

The refreshed `packages/ICDE_TRAPS_github_code.zip` should be uploaded to the artifact service for submission, and the resulting artifact URL should be entered in the submission system.
