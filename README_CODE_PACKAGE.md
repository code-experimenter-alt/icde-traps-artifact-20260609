# TRAPS ICDE Submission Package

This package contains the LaTeX source, generated figures, and local experiment scripts for the TRAPS ICDE submission.

## Paper Build

Build the paper from the repository root:

```powershell
latexmk -pdf -interaction=nonstopmode -halt-on-error main.tex
```

No appendix is included in the submission source or compiled PDF.

## Experiment Figures

The ICDE-oriented figures are generated by:

```powershell
python experiments/run_icde_experiments.py
```

Outputs:

- `figure/icde_experiments/*.pdf`
- `figure/icde_experiments/*.png`
- `experiments/results/*.csv`

The script is deterministic and uses fixed seeds for the controlled admission-index, shift-grid, dynamic drift-guard, ablation, update, queueing, and replay-cache visualizations. Individual suites can also be run, for example:

```powershell
python experiments/run_icde_experiments.py --suite frontier_production --seeds 0-29 --bits 4,6,8,10,12,14,16,20,24
python experiments/run_icde_experiments.py --suite shift_grid --seeds 0-9 --bits 12,16
python experiments/run_icde_experiments.py --suite traps_ablation --seeds 0-29 --bits 12,16
```

The `shift_grid` suite also writes `experiments/results/shift_dynamic_capacity.csv` and `shift_dynamic_capacity_summary.csv`, which evaluate frozen, next-epoch dynamic, and drift-guard allocation using previous-window region observations without future labels.

The LANL replay downloads the public LANL authentication trace and writes two evidence views: a status-outcome replay using the trace's real `Success`/`Fail` status labels, and an invalid-heavy trace-conditioned stress replay using generated password-validity labels:

```powershell
$env:LANL_AUTH_INSECURE='1'  # only needed if local Python cannot verify the LANL TLS chain
python experiments/run_lanl_trace_experiment.py
```

Run the local submission audit after regeneration:

```powershell
python experiments/package_submission.py
python experiments/audit_submission_requirements.py
```

Expected result: packaging refreshes `packages/ICDE_TRAPS_github_code.zip`, `packages/ICDE_TRAPS_overleaf_minimal.zip`, and `packages/SHA256SUMS.txt`; every audit line then prints `PASS`, followed by `All submission requirement audits passed.`

## Artifact Status

The current main paper does not depend on old dashboard PDFs or split variants. The ICDE-facing figures are generated locally from the scripts and CSV outputs included in this package.

## Notes

- `ARTIFACT_EVALUATION.md` gives a reviewer-facing quick-start checklist.
- `PACKAGE_MANIFEST.md` records the expected submission files, package contents, and upload requirement.
- `SUBMISSION_ARTIFACT_UPLOAD.md` gives the artifact upload checklist and suggested submission metadata.
- Build byproducts such as `.aux`, `.log`, `.fls`, and `.fdb_latexmk` are intentionally excluded from the GitHub package.
