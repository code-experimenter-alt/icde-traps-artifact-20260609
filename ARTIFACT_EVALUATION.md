# TRAPS Artifact Evaluation Guide

This artifact supports the ICDE submission of TRAPS. It contains the paper source, deterministic controlled experiments, LANL-conditioned replay code, generated CSV outputs, generated figures, and an executable audit for the submission requirements.

## Requirements

- Windows PowerShell or a compatible shell.
- Python 3.12 with `numpy`, `matplotlib`, and `scikit-learn`.
- MiKTeX/LaTeX with `latexmk`.
- Network access only for the LANL-conditioned replay script.

## Build the Paper

```powershell
latexmk -pdf -interaction=nonstopmode -halt-on-error main.tex
```

Expected result:

- `main.pdf`
- 13 pages total in the current build. The 12-page main body ends before the AI-generated content acknowledgement and references on page 13.
- No fatal errors, undefined references/citations, or overfull hboxes in `main.log`.

## Regenerate Controlled Experiments

```powershell
python experiments/run_icde_experiments.py
```

This regenerates:

- `experiments/results/frontier_production.csv`
- `experiments/results/frontier_production_summary.csv`
- `experiments/results/frontier_production_memory.csv`
- `experiments/results/shift_grid.csv`
- `experiments/results/shift_grid_sensitivity.csv`
- `experiments/results/shift_grid_worstcase.csv`
- `experiments/results/shift_dynamic_capacity.csv`
- `experiments/results/shift_dynamic_capacity_summary.csv`
- `experiments/results/traps_ablation.csv`
- `experiments/results/queueing_by_method.csv`
- `figure/icde_experiments/icde_frontier_production.pdf`
- `figure/icde_experiments/icde_shift_grid_heatmap.pdf`
- `figure/icde_experiments/icde_queueing_by_method.pdf`

The script also supports suite-level execution:

```powershell
python experiments/run_icde_experiments.py --suite frontier_production --seeds 0-29 --bits 4,6,8,10,12,14,16,20,24
python experiments/run_icde_experiments.py --suite shift_grid --seeds 0-9 --bits 12,16
python experiments/run_icde_experiments.py --suite traps_ablation --seeds 0-29 --bits 12,16
```

The dynamic-capacity rows are generated by the `shift_grid` suite. They compare frozen, next-epoch dynamic, and drift-guard allocation; the audit requires no future-label use, zero represented-valid rejects, and a guard worst case near the Global Bloom fallback.

## Regenerate LANL-Conditioned Replay

```powershell
$env:LANL_AUTH_INSECURE='1'
python experiments/run_lanl_trace_experiment.py
```

The environment variable is needed only on machines whose Python certificate store cannot verify the LANL TLS chain. The script writes a status-outcome replay using LANL `Success`/`Fail` labels, plus an invalid-heavy trace-conditioned stress replay whose password-validity labels are generated locally by the disclosed model.

Expected outputs:

- `experiments/results/lanl_trace_conditioned.csv`
- `experiments/results/lanl_trace_denominators.csv`
- `experiments/results/lanl_trace_negative_controls.csv`
- `experiments/results/lanl_status_outcome.csv`
- `experiments/results/lanl_status_outcome_denominators.csv`
- `figure/icde_experiments/icde_lanl_trace_suite.pdf`

## Run the Requirement Audit

```powershell
python experiments/package_submission.py
python experiments/audit_submission_requirements.py
```

Expected result:

```text
PASS audit_frontier
PASS audit_shift_grid
PASS audit_ablation
PASS audit_lanl
PASS audit_queueing
PASS audit_manuscript_framing
PASS audit_latex_and_packages
All submission requirement audits passed.
```

The audit checks the required method sets, seed and bits/key grids, dynamic drift-guard evidence, denominators, cache invariants, LANL negative controls, queueing outputs, manuscript framing/overclaim guards, LaTeX log quality, page count, AI disclosure, font hygiene, package deliverables, PDF/zip/source hygiene for the current author mode, and whether critical files inside the GitHub and Overleaf zips match the current worktree.
The packaging step also writes `packages/SHA256SUMS.txt`; the audit verifies that the recorded SHA-256 hashes still match `main.pdf` and both submission zip files.
